# Contributing to ILC.Fishbowl.JS

## Reporting Issues

When reporting an issue, please include:

- **Description** — Clear summary of the problem or unexpected behavior
- **Steps to reproduce** — Minimal code or configuration to trigger the issue
- **Expected vs actual behavior** — What you expected and what happened instead
- **Fishbowl version** — The Fishbowl Advanced version you are running (e.g., 26.1)
- **Environment** — JXBrowser, Web, or Demo mode

**Important:** Issues must also be reported to your ILC Technology support contact. GitHub issues alone may not be monitored in real time. Issues are prioritized based on impact, available resources, and customer-specific SLAs.

## Pull Requests

Pull requests are welcome but must meet the following requirements:

1. **Link to an issue** — Every PR must reference an existing issue that describes the bug or problem being fixed. If no issue exists, open one first.

2. **Tested against a recent Fishbowl version** — Changes must be tested against a supported version of Fishbowl Advanced. Include the version number in your PR description.

3. **Clear documentation** — The PR description must explain:
   - What problem the change fixes (reference the issue number)
   - How the fix works
   - Any side effects or breaking changes

4. **No scope creep** — PRs should address a single issue. Avoid bundling unrelated changes, refactors, or style fixes.

5. **Preserve compatibility** — Changes must not break existing CloudPages that depend on current behavior. If a breaking change is necessary, document it clearly and explain why.

Pull requests that do not meet these requirements will be asked to revise before review. Acceptance is at the discretion of ILC Technology LLC.
